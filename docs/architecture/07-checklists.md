# 07 — Чеклисты

> Оглавление: [`README.md`](README.md).

Перед завершением используйте чеклист под тип задачи. Агент **MUST** пройти релевантный чеклист и устранить все несоответствия.

---

## A. Новая страница

- [ ] Слайс лежит в `pages/<page-name>/` (kebab-case).
- [ ] Файл `index.ts` страницы существует и ре-экспортирует только то, что нужно `app` (обычно компонент страницы).
- [ ] Код разнесён по сегментам (`ui`, `model`, при необходимости `api`, `lib`, `config`).
- [ ] Локальные для страницы UI-блоки, которые не переиспользуются, **остаются** в слайсе страницы (pages-first).
- [ ] Маршрут привязан из `app/` (или `pages/<page>` экспортирует определение маршрута, используемое `app`). Детали — [`09-routing.md`](09-routing.md).
- [ ] Нет импортов из соседних `pages/*`.
- [ ] Нет глубоких импортов во внутренности других слайсов.
- [ ] Все межслайсовые импорты — через абсолютные алиасы.

---

## B. Новый виджет

- [ ] Экстракция оправдана **либо** переиспользованием на ≥ 2 страницах, **либо** тем, что блок — крупная независимая область одной страницы (v2.1-исключение).
- [ ] Слайс лежит в `widgets/<name>/`.
- [ ] `index.ts` даёт явные именованные экспорты (без wildcards).
- [ ] Сегменты: `ui`, `model`, при необходимости `api` / `lib` / `config`.
- [ ] Потребители импортируют только из `@/widgets/<name>` (не из глубины).
- [ ] Нет импортов вышестоящего слоя (`pages`, `app`).
- [ ] Нет импорта другого виджета того же слоя.

---

## C. Новая фича

- [ ] Фича представляет переиспользуемое, бизнес-значимое взаимодействие.
- [ ] Слайс лежит в `features/<name>/`, имя в kebab-case (обычно глагольное: `add-to-cart`, `like-post`).
- [ ] `index.ts` даёт явные именованные экспорты.
- [ ] Сегменты: `ui`, `model`, при необходимости `api` / `lib` / `config`.
- [ ] Нет импорта `widgets`, `pages`, `app`.
- [ ] Нет импорта другой фичи того же слоя.

---

## D. Новая сущность

- [ ] Слайс лежит в `entities/<name>/` и именует доменный концепт.
- [ ] `index.ts` существует с явными именованными экспортами.
- [ ] Доменные типы — в `model/` (не в корне слайса).
- [ ] Если сущность ссылается на другую: создан `@x/<consumer>.ts` в **экспортёре**, потребитель импортирует оттуда.
- [ ] Нет обычного импорта entity того же слоя (только через `@x`).
- [ ] Нет импорта вышестоящего слоя.

---

## E. Новая утилита/примитив в shared

- [ ] Код проект-независим, не содержит бизнес-знаний.
- [ ] Размещён в правильном сегменте:
  - UI-примитив → `shared/ui/<name>/`
  - Утилита → `shared/lib/<purpose>/`
  - Транспорт → `shared/api/`
  - Конфиг / флаги → `shared/config/`
- [ ] Для примитивов `shared/ui`: у каждого — своя папка с `index.ts`.
- [ ] Нет импортов из слоёв выше `shared`.

---

## F. Рефакторинг (перенос кода между слоями)

- [ ] Целевая локация выбрана по pages-first (см. [05-pages-first.md](05-pages-first.md)).
- [ ] У целевого слайса после переноса есть `index.ts` с явными именованными экспортами.
- [ ] Все потребители теперь импортируют из Public API целевого слайса (без глубоких путей).
- [ ] В исходной локации перенесённого кода больше нет.
- [ ] Побочно не появились кросс-импорты одного слоя.
- [ ] Если появилась новая кросс-ссылка между entities: добавлен `@x`-фасад вместо прямого импорта.
- [ ] Локально проходят тайпчек и тесты.

---

## G. Добавление кросс-ссылки между entities

- [ ] Подтверждено, что вынос в `shared` или композиция с более высокого слоя не подходит.
- [ ] Подтверждено, что обе сущности — действительно разные доменные концепты (иначе — слить).
- [ ] Создан `entities/<exporter>/@x/<consumer>.ts`, ре-экспортирующий только необходимые потребителю символы.
- [ ] В ре-экспорт попали только типы и небольшие read-only хелперы (без UI, без stores).
- [ ] Потребитель импортирует исключительно из `@/entities/<exporter>/@x/<consumer>`.
- [ ] Обычный импорт `@/entities/<exporter>` на стороне потребителя удалён.

---

## H. Лифт-ап: извлечение фичи из сущности

Применяется, когда код из `entities/<name>/ui/` перерастает роль «показать сущность» и превращается в интерактивную композицию. Типовые триггеры:

- Runtime cross-entity: файл импортирует хук/store/сервис другой entity ([04-cross-imports.md](04-cross-imports.md#runtime-cross-entity-это-features-не-x)).
- Переиспользование на ≥ 2 страницах/виджетах.
- Самодостаточная cross-entity область ≥ ~200 строк даже в одном потребителе.

### Чек-лист

- [ ] Подтверждён как минимум один из триггеров выше. «Большой файл» сам по себе не повод — маленькие одноразовые формы остаются в pages / entities без экстракции.
- [ ] Создан `features/<verb>-<name>/` (kebab-case, глагольное имя: `manage-product`, `add-to-cart`).
- [ ] `features/<name>/index.ts` существует с явными именованными ре-экспортами.
- [ ] Из `entities/<name>/index.ts` **удалены** имена, перенесённые в фичу (Public API сущности сокращается).
- [ ] Внутри перенесённого файла пути `'../model/types'`, `'../lib/*'` переведены на `@/entities/<name>` (фича **MUST** работать через Public API сущности, а не через её внутренности).
- [ ] Сиблинг-импорты (`'./margin-display'`) остаются относительными **только** если сосед тоже переехал в эту фичу. Иначе → `@/entities/<name>` или `@/features/<other>`.
- [ ] Все потребители переключены: `grep -rn "<OldName>" src/` → замена на `@/features/<name>`.
- [ ] В исходной локации (`entities/<name>/ui/`) файла с тем же именем больше нет (проверить `find`).
- [ ] Тесты экстрагированного компонента перенесены вместе с кодом.
- [ ] Тайпчек и тесты проходят локально.

### Сигналы, что экстракция правильная

- `entities/<name>/index.ts` после экстракции содержит **только** display-примитивы (`Card`, `List`, `Details`) — «как показать сущность».
- `features/<name>/index.ts` содержит **только** интерактивные компоненты — «как действовать с сущностью».
- Ни одна фича не ре-экспортирует тип сущности — типы живут в entity, фича использует их через Public API.

## I. Архитектурный pre-flight (на каждый коммит)

Короткий архитектурный чек-лист перед коммитом. Повторяет ключевые инварианты в одном месте.

- [ ] Нет сегментов, названных по сущности (`components`, `hooks`, `types`, `utils`, `helpers`).
- [ ] Нет wildcard-ре-экспортов (`export * from "..."`) ни в одном `index.ts`.
- [ ] Нет глубоких импортов в обход `index.ts` слайса.
- [ ] Нет импортов «снизу вверх» по слоям.
- [ ] У каждого нового слайса есть `index.ts`.
- [ ] Не заведено пустых «на будущее» слайсов.

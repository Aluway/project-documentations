## Что такое FSD?

**Feature-Sliced Design (FSD)** — архитектурная методология для структурирования фронтенд-приложений. Она стандартизирует организацию кода: разделение проекта по слоям, слоёв — по бизнес-доменам (слайсам), а слайсов — по техническим сегментам.

Цель: стабильная, удобная для рефакторинга, унифицированная кодовая база, устойчивая к изменению бизнес-требований.

## Почему v2.1?

FSD 2.1 заменяет v2.0-подход «сначала сущности и фичи» на **pages-first**:

> Держите большую часть UI и логики на странице, где она используется. Переносите код в нижний слой только когда появляется конкретный случай переиспользования.

Следствия:
- Ниже порог входа — естественная отправная точка всегда страница.
- Меньше преждевременных абстракций — пустые «на будущее» сущности и фичи не приветствуются.
- Виджеты теперь **MAY** владеть собственным состоянием, API и бизнес-логикой (больше не только композиционный слой).
- Кросс-импорты между сущностями стандартизированы через нотацию `@x` (Public API for cross-imports).

v2.1 **не ломает совместимость** — существующий v2.0-код остаётся валидным. **Новый код MUST следовать v2.1.**

Руководство описывает шесть активных слоёв: `app`, `pages`, `widgets`, `features`, `entities`, `shared`. Более ранний слой `processes` **deprecated** и **MUST NOT** появляться в новом коде. Подробности: [01-layers.md](01-layers.md).

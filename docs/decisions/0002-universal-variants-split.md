# ADR 0002: Разделение code-style на `universal/` + `variants/` через `PROFILE.md`

> Оглавление: [`README.md`](README.md).

---

## Status

accepted

---

## Context

React-экосистема содержит множество альтернативных инструментов, решающих одну задачу: TanStack Query vs SWR vs RTK Query для серверного state; Zustand vs Redux Toolkit vs Jotai для глобального клиентского; React Hook Form vs Formik vs React 19 Actions для форм; Tailwind vs CSS Modules vs styled-components для стилей. Документация style-guide, написанная под один стек, неприменима к другому.

При этом значительная часть правил **не зависит от стека**: TypeScript `strict`, Rules of Hooks, semantic HTML, приоритет queries в тестах, behavior-first подход. Эти правила одинаково верны во всех React+TS проектах.

Нужна схема, в которой:

1. Общие правила пишутся один раз и применяются ко всем проектам.
2. Правила, зависящие от инструмента, изолируются и активируются избирательно.
3. Агент получает однозначный ответ, какие правила активны в **этом** репозитории.
4. Замена инструмента (миграция TanStack Query → SWR) не требует переписывания универсальных правил.

---

## Decision

Принято следующее:

- Каталог `docs/code-style/` **MUST** делиться на `universal/` и `variants/`:
  - `universal/*.md` — правила, применяющиеся **всегда**, независимо от стека. Применяются в любом React+TS проекте.
  - `variants/*.md` — правила, применяющиеся **условно**, при активации соответствующего поля в `PROFILE.md`.
- В корне `docs/code-style/` **MUST** лежать `PROFILE.md` — источник истины о стеке данного репозитория. Он фиксирует версии зависимостей, выбранные инструменты и список активных variant-глав.
- Агент **MUST** читать `PROFILE.md` **первым** при работе в области code-style и активировать только variant-главы, отмеченные `[x]` в «Active Variant Modules».
- Variant-глава **MUST** содержать в frontmatter поле `requires`, совпадающее с полем/значением в `PROFILE.md` (формат — [`../_meta/frontmatter.md`](../_meta/frontmatter.md) раздел 3).
- Если `PROFILE.md` не заполнен — применяется **fallback-стратегия** из [`../code-style/PROFILE.md`](../code-style/PROFILE.md) секции «Fallback-стратегия»: universal + consensus-дефолт 2026 + пометка вывода как требующего ревью.
- Конфликтующие сочетания variant'ов фиксируются в [`../code-style/PROFILE.md`](../code-style/PROFILE.md) «Compatibility Matrix» и **MUST** проверяться перед началом задачи.
- Legacy-инструменты (Formik, styled-components, Enzyme, CRA) **MUST NOT** иметь собственных variant-глав. Для них применяется только universal-минимум, и каждая нетривиальная правка **SHOULD** сопровождаться ADR о миграции.

Паттерн `universal/` + `variants/` + `PROFILE.md` **MAY** переиспользоваться другими категориями, где возникает та же проблема «общие правила vs выбор инструмента».

---

## Consequences

### Positive

- Общие правила пишутся один раз — снижается объём документации и риск рассинхрона.
- Смена инструмента — локальное изменение: правка `PROFILE.md` + подключение другого variant'а, без переписывания универсальных правил.
- Агент получает предельно конкретный scope: он знает, какие главы активны, и не применяет правила неактивных инструментов.
- Compatibility Matrix делает несовместимые сочетания видимыми до начала работы, а не при ревью.

### Negative

- Двойной layer добавляет когнитивную нагрузку для человека-автора: нужно решать, в какой слой положить правило.
- Правило, которое сейчас универсально, может оказаться стек-зависимым при добавлении нового variant'а — требуется рефакторинг.
- Fallback-стратегия для незаполненного `PROFILE.md` — компромисс; в автономном режиме агент может применить дефолты, не подходящие проекту.

### Neutral

- Количество variant-глав растёт с экосистемой. Нужен плановый пересмотр — см. [`../_meta/governance.md`](../_meta/governance.md) раздел 2.

---

## Alternatives considered

- **Один большой style-guide без разделения.** Отклонено: писать «если у вас TanStack Query, то…; если SWR, то…» на каждом правиле — неподдерживаемо; агент теряется в условностях.
- **Отдельный style-guide на каждый стек** (style-guide-tanstack.md, style-guide-redux.md). Отклонено: массовое дублирование общих правил; правка общего правила — правка во всех файлах; рассинхрон неизбежен.
- **Детектирование стека по `package.json` вместо явного `PROFILE.md`.** Отклонено: `package.json` отражает установленные пакеты, но не выбор («какой из двух используется»); неявность ломает автоматизацию; `PROFILE.md` явно читается человеком и агентом.
- **Variant-правила как условия внутри универсальных глав** (`MUST X, если используется Y`). Отклонено: универсальная глава перегружается условностями; навигация по профилю усложняется; правило применимости variant'а становится не машино-проверяемым.

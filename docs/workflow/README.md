# Workflow — процесс разработки

> Точка входа для агента: [`AGENTS.md`](AGENTS.md). Обоснование категории: [ADR 0007](../decisions/0007-introduce-workflow-category.md).

Процессные правила, не зависящие от стека: git, PR, code-review, релизы. Применяются ко всем репозиториям-потомкам этого шаблона, если иное не зафиксировано override-ADR в конкретном проекте.

---

## Зачем отдельная категория

До введения этой категории процессные правила жили в двух разных местах:

- В корневом `AGENTS.md` — короткие упоминания без деталей («Commit message follows `<type>(<scope>): <subject>` convention»).
- В глобальном `CLAUDE.md` пользователя — полные правила, не привязанные к репо и не переносимые при форке.

Категория собирает их в одном месте, версионирует, линтует и делает обнаруживаемыми через Document Map. Подробности — [ADR 0007](../decisions/0007-introduce-workflow-category.md).

## Принципы, пронизывающие категорию

- **Атомарность > удобство.** Один коммит — одна мысль. Смешанные коммиты затрудняют ревью, откат и `git bisect`.
- **Переписывать локально, сохранять публично.** До открытия PR история — черновик. После — неизменяема.
- **Маленький PR побеждает большой.** Диффы ≤ 500 строк ревьюятся в разы быстрее и с меньшим числом пропущенных дефектов.
- **Явное > неявное.** Тип коммита (`feat`/`fix`/`refactor`) и тип PR (`feat`/`fix`) проговариваются в заголовке, не угадываются по содержимому.
- **Семантические версии для кода, дата-версии для документации.** Semver — для релизов продукта; документация версионируется per-глава (см. [`../_meta/frontmatter.md`](../_meta/frontmatter.md) раздел 2).

## Как переопределить в форке

Если проект-потомок требует другого процесса (например, trunk-based вместо release-branch, squash-merge vместо rebase-merge):

1. Открыть ADR в `docs/decisions/` с указанием `supersedes: <id главы или принципа>`.
2. Обновить соответствующую главу в `docs/workflow/` либо полностью заменить её содержимым под новый процесс.
3. Обновить [`AGENTS.md`](AGENTS.md) — Hard Invariants и Pre-Flight.
4. Закоммитить одним PR.

Проект-потомок **MUST** либо наследовать правила template-репо, либо явно override-нуть их через ADR. «Правило неформально игнорируется» — не допускается.

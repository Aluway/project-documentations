# Onboarding — адопция template'а

> Точка входа для агента: [`AGENTS.md`](AGENTS.md). Обоснование категории: [ADR 0010](../decisions/0010-introduce-onboarding-category.md).

Категория для **первого дня после форка** template-репозитория. Не нужна постоянно; читается один раз при каждой адопции. Целевой читатель — разработчик или ИИ-агент, только что склонировавший шаблон в новый проект.

---

## Зачем отдельная категория

До введения этой категории первые шаги после форка описывались только в корневом [`../../README.md`](../../README.md) — 5-пунктовой секцией «Using this template». Этого было недостаточно:

- Нет пошагового чек-листа по заполнению каждого поля `PROFILE.md`.
- Нет инструкции, что делать с template-версиями `README.md` и `CHANGELOG.md`.
- Нет разветвлений под типовые сценарии (монорепо, fullstack, Expo, SSR-first).
- Нет red-flag'ов: «если X — остановитесь и обсудите с командой».

Эта категория восполняет пробел. Корневой README остаётся кратким landing'ом, onboarding — детальным чек-листом. Обоснование — [ADR 0010](../decisions/0010-introduce-onboarding-category.md).

## Содержание

1. [`01-first-fork.md`](01-first-fork.md) — Чек-лист первого дня после форка: от клонирования до первого коммита с зелёным линтером. ~15 шагов.
2. [`02-common-branches.md`](02-common-branches.md) — Разветвления: монорепо, fullstack с backend, Expo/Electron, Next.js SSR-first, solo-репо без CI. Для каждого сценария — список override'ов и добавлений.

## Что читать после онбординга

Когда `01-first-fork.md` пройден и первый коммит сделан:

1. [`../workflow/AGENTS.md`](../workflow/AGENTS.md) — git-процесс, PR, review, релизы. Применяется на каждую задачу.
2. [`../architecture/AGENTS.md`](../architecture/AGENTS.md) — FSD 2.1 pages-first. Открывается при каждом размещении кода.
3. [`../code-style/AGENTS.md`](../code-style/AGENTS.md) — React+TS. Открывается с PROFILE в руках.
4. [`../security/AGENTS.md`](../security/AGENTS.md) — secrets, input handling, deps. Открывается при соответствующих задачах.
5. [`../decisions/AGENTS.md`](../decisions/AGENTS.md) — ADR. Открывается при необходимости принять структурное решение.
6. [`../_meta/AGENTS.md`](../_meta/AGENTS.md) — правка самой документации. Открывается редко.

После онбординга эта категория **MAY** не открываться до следующего форка.

## Принципы, пронизывающие категорию

- **Проверяемые шаги, не абстракции.** Каждый пункт чек-листа — конкретное действие (заполнить поле, создать файл, запустить команду). «Изучить принципы» — не шаг.
- **Последовательность важна.** Шаги зависят друг от друга; перескок ломает флоу. Чек-лист **SHOULD** проходиться сверху вниз.
- **Развилки явны.** Каждое отличие от дефолта (монорепо, SSR, mobile) документируется как отдельная ветка в `02-common-branches.md`, не размывает основной чек-лист.
- **Успех измеряется линтером.** Зелёный `node scripts/lint-docs.mjs` после онбординга — объективный критерий завершения.

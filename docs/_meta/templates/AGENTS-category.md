# Template — `AGENTS.md` категории

> Оглавление: [`../AGENTS.md`](../AGENTS.md). Стиль: [`../style-guide.md`](../style-guide.md).

Используйте этот шаблон при создании новой категории в `docs/<new-category>/`. Скопируйте содержимое ниже (всё, что между `--- 8< ---` маркерами), вставьте в `docs/<new-category>/AGENTS.md`, заполните плейсхолдеры в `<угловых скобках>`.

Обязательные секции: Scope, When to Read, Working Protocol, Hard Invariants, Pre-Flight Checklist, «If Chapters Disagree with This File». Удалять их **MUST NOT**. Переименовывать — **MUST NOT**.

Порядок секций канонический; **MUST** соблюдаться, **за одним исключением**: если категория использует условную активацию глав через PROFILE-поле (как [`../../code-style/`](../../code-style/AGENTS.md)), Working Protocol **MAY** идти перед When to Read — Working Protocol в таких категориях начинается с «прочитайте PROFILE, решите, какие главы применимы», и When to Read естественно идёт после. Отклонение **MUST** быть очевидно из содержимого Working Protocol; ADR не требуется.

## Что заполняется после копирования

- `version` — начать с `1.0.0`.
- `last-reviewed` — сегодняшняя дата в формате `YYYY-MM-DD`.
- `<Название категории>` — человеко-читаемое имя в верхнем заголовке.
- `Scope` — одно предложение + 3–5 буллетов. Явно укажите, что **не** покрывает.
- `When to Read` — таблица со всеми главами категории, ровно одна строка на главу.
- `Working Protocol` — последовательность 4–6 шагов. Каждый шаг — ссылка на конкретную главу.
- `Hard Invariants` — 5–10 инвариантов, каждый с `MUST` / `MUST NOT` / `SHOULD` и ссылкой на главу, где это правило подробно описано.
- `Pre-Flight Checklist` — 7–15 проверяемых утверждений в форме результата.

## После создания категории

1. Создайте `docs/<category>/README.md` — описательное оглавление (см. как сделано в [`../../architecture/README.md`](../../architecture/README.md) и [`../../code-style/README.md`](../../code-style/README.md)).
2. Обновите корневой [`../../../AGENTS.md`](../../../AGENTS.md) — добавьте строку в Document Map.
3. Откройте ADR в [`../../decisions/`](../../decisions/AGENTS.md) — «Introduce category <name>».
4. Прогоните Pre-Flight из [`../AGENTS.md`](../AGENTS.md).
